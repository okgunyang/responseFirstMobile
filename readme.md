## 크로스플랫폼 앱 개발 도구
안드로이드나 iOS, Window 기반의 앱을 모두 개발하거나 이미 개발된 앱을 다른 플랫폼에 맞게 바꾸어 주는 개발 도구
1. [Flutter](https://flutter-ko.dev)
    - 플러터는 구글이 만든 캔버스라는 객체와 UI 컴포넌트를 이용해서 화면을 그려 주는 IDE이면서 언어이다.
2. [React Native](https://reactnative.dev)
    - 리액트 네이티브(React Native)는 페이스북이 개발한 오픈 소스 모바일 애플리케이션 프레임워크로서 크로스플랫폼 IDE이다.
3. [Cordova](https://cordova.apache.org)
    - Nitobi라는 회사에서 처음 개발한 PhoneGap의 진화된 버전
4. [Xamarin](https://visualstudio.microsoft.com/ko/vs/)
    - Microsoft에서 개발한 크로스플랫폼 IDE
5. [PWA](https://web.dev/progressive-web-apps/#introduction)
    - Progressive Web Apps(PWA)는 HTML, CSS, 자바스크립트를 사용하여 개발하는 방식으로서 IDE라기보다는 기존 기술에 앱으로 전환가능하도록하는 방법을 의미한다.
6. [Ionic](https://ionic.io/)
    - 고품질의 크로스 플랫폼 네이티브 및 웹 앱 경험을 구축하기 위한 오픈 소스 모바일 UI 툴킷으로서 자바스크립트를 기반으로 앱을 구축할 수 있도록한 개발 통합 환경을 제공한다.
7. 기타
    - 그 밖에 Perfecto Mobile(퍼펙토 모바일), Kony Visualizer(코니 비주얼라이저), AppInstitute(앱연구소), Alpha Anywhere(알파 애니웨어), Kinvey(킨비), Mobincube(모빈큐브), Longrange(롱레인지), Appian Quick Apps(아피안 퀵 앱), Appypie(앱 파이), AppMaker(앱 메이커), Kobiton(코비톤), Buddy(버디), Appery(앱퍼리), AppMaster(앱마스터) 등이 있다.


## 프로그레시브 웹 앱(PWA:Progressive Web App) 프론트 엔드 프레임워크
- 자바스크립트 프레임워크를 활용하여 모바일 앱을 개발하는 방식으로 크로스플랫폼 IDE로 여러 플랫폼의 하이브리드 앱이나 웹 앱을 제작할 수 있으며, 하나의 문서에 여러 웹 페이지를 교대로 띄우는 SPA(Sing Page Application)를 기본적으로 구현한다.
1. React 
    - SPA(Sing Page Application) 방식 위주의 자바스크립트 웹 프레임워크로서 TypeScript나 Sass 등 도 지원한다.
    - [사이트 바로가기](https://ko.reactjs.org)
2. Angular
    - 구글에서 운용, 유지 중인 자바스크립트 웹 프레임워크로서 기본적으로 TypeScript를 기반으로 하여 제작한다.
    - [사이트 바로가기](https://angularjs.org)
3. Vue
    - 구글에서 Angular를 제작하던 Evan You에 의해 개발한 자바스크립트 웹 프레임워크로서 라우팅, 상태 관리, 빌드 도구화와 같은 여러 고급 기능을 지원하며, TypeScript나 Sass 등 도 지원한다.
    - [사이트 바로가기](https://vuejs.org)
4. Ember
    - Ember 코어 팀이 개발한 자바스크립트 웹 프레임워크로서 Routes, Models, Templates, Components, Services 등을 활용하여 앱을 제작한다.      
    - [사이트 바로가기](https://emberjs.com)
5. Nuxt
    - vuejs 기반으로 서버사이드 렌더링 웹페이지 구현이 가능하게 하는 라이브러리로 특정 프레임워크에 의존적이어서 프레임워크보다는 라이브러리에 가깝다.
    - [사이트 바로가기](https://nuxtjs.org)
6. Backbone
    - 이벤트 기반을 위해 2010년 Jeremy Ashkenas가 만든 미니멀리스트 자바스크립트 우ㅞㅂ 프레임워크으로서 JavaScript 기능과 모듈을 JavaScript 코드의 빌딩 블록으로 사용하고 모듈 바인딩 및 사용자 지정 발생을 제공한다.
    - [사이트 바로가기](https://backbonejs.org)
7. NextJS
    - React로 정적 웹 애플리케이션을 구축하기 위한 무료 오픈 소스 프레임워크
    - [사이트 바로가기](https://nextjs.org)


## 자바스크립트 웹 서버 프로그래밍 백엔드 프레임워크
1. Node
    - 자바스크립트 언어를 활용하여 웹 프로그래밍된 문서를 웹 브라우저에서 실행되도록 하는 런타임환경을 제공한다.
    - [사이트 바로가기](https://nodejs.org/ko)
2. Express
    - Node에서 동작하는 서버 프레임워크이므로 우선 Node를 설치해야하며, 기존 Node보다 훨씬 빠른 속도를 제공하고, 사용되는 미들웨어로는 프론트엔드 프레임워크가 React인 경우 Nextjs를 사용, Vue인 경우 Nuxtjs를 활용한다.
    - [사이트 바로가기](https://expressjs.com)
3. Nest
    - Express가 우선 설치하고, 그러고 난 후 활용가능한 프레임워크로서 프론트엔드 프레임워크는 Angular를 기본으로 활용해야 한다.
    - [사이트 바로가기](https://nestjs.com/)



## 자바스크립트 방식의 언어
1. ECMA Script
    - 넷스케이프에서 개발한 자바스크립트의 표준 문법을 따르도록 하는 자바스크립트 기본 언어로서 Node를 활용하여 서버 프로그래밍도 가능하다.
2. TypeScript
    - 마이크로소프트에서 발표한 JavaScript에 정적 타입 개념을 추가한 신형 언어로서 자바스크립트와 가장 많이 다른 점을 보이는 것은 자료형(Type)을 구분하는 것에 있다.
3. CoffeeScript
    - JavaScript의 문법을 개선하여 Jeremy Ashkenas가 개발한 신형 언어로서 자바스크립트와 가장 많이 다른 점은 람다식과 같은 표현으로 함수나 조건문 등의 블록이나 선언을 사용하는 것에 있다.
4. VanillaScript
    - 외부 라이브러리나 프레임워크를 사용하지 않고 순수 자바스크립트만으로 구현하는 것을 말한다.
5. Dart
    - 구글에서 발표한 JavaScript 문법을 개선한 언어로서 기존의 언어보다 간결하고 강력한 기능을 제공하며, Dart VM 엔진상에서 동작한다.


## CSS 프레임워크
스타일을 사용자가 모두 직접 지정하거나 코딩하려면, 엄청난 시간이 소요되므로, 기본언어인 html을 작성하고, 태그에 해당 속성이나 클래스 등을 부여하면, 자동으로 스타일이 부여되는 프레임워크로서 대부분의 CSS 프레임워크는 link 요소로 css 파일을 연결하면, 적용될 수 있다.
1. Bootstrap
    - 트위터에서 시작된 오픈 소스 프론트엔드 프레임워크. 시작은 디자이너 한명과 트위터의 한 개발자였지만 지금은 트위터에서 주도적으로 개발하고 있지는 않다. 트위터에서 사용하는 각종 레이아웃, 버튼, 입력창 등의 디자인과 기능을 CSS와 JavaScript로 만들어 놓은 것
    - [사이트 바로가기](https://getbootstrap.kr)

2. Tailwind
    - Adam Wathan이 고안하고, Tailwind Labs에서 개발하고 있는 CSS 프레임워크로서 해당 CSS나 JS 파일을 CDN으로 연결하지 않고, 설치하거나 해당 파일을 다운로드 받아 사용하는 방식을 활용하는 프레임워크이다.
    - [사이트 바로가기](https://tailwindcss.com)

3. Bulma
    - Flexbox 기반의 CSS 프레임워크로서 부트스트랩처럼 JS에 의해 동작하는 부분이 없어 동적인 스크립트를 별도로 구축해야 하는 단점이 있기는 하나 상당히 가볍고, CSS3 Level에 잘 작동한다.
    - [사이트 바로가기](https://bulma.io)

4. Foundation
    - Zurb사에서 만들었으며, 내장 구성요소가 화려한 것이 특징이며, 이메일 디자인, 수많은 애니메이션이 제공되는 모션 UI 등이 화려하다.
    - [사이트 바로가기](https://get.foundation)

5. Pure
    - yahoo에서 개발하였으며, 경량 프레임워크로서 비교적 디자인이 심플하고, CSS를 모듈화하여 활용할 때 훨씬 효과적이고, 다소 사용자화가 간단한 편이다.
    - [사이트 바로가기](https://purecss.io)

6. UI Kit
    - 복잡한 프론트엔드 레이아웃을 주로 구현할 때 유리한 프레임워크로 Sass와 Less로 전처리하는 기능도 쉽게 사용할 수 있게 하였으며, 쉬운 API와 깔끔한 디자인이 특징이다.
    - [사이트 바로가기](https://getuikit.com)

7. Milligram
    - 이름 만큼 상당한 작은 2KB 크기에 매우 가볍고 배우기 쉽지만, 다른 프레임워크처럼 기본 스타일을 제공하지 않는 것이 특징이다.
    - [사이트 바로가기](https://milligram.io)  

8. Tachons
    - Tachyons는 생산성을 높이기 위해 뛰어난 유틸리티 클래스를 제공하는 데 중점을 두고 있으며, HTML, Rails, React, Angular 등과 같은 다양한 설정에서 사용할 수 있는 기능 템플릿을 제공하는 것이 특징이다.
    - [사이트 바로가기](https://tachyons.io)

9. Materialise

## Style(스타일) PreProcessor(전처리기)
순수 CSS에 없는 기능인 반복 처리, 조건 분기 처리, 함수나 변수의 정의 등이 대표적이며, 이러한 CSS에 없는 기능을 활용하여 보다 빠르고 효율적으로 스타일을 적용할 수 있도록 하고, 또한 전처리기로 작성된 문서는 다시 CSS로 컴파일하여 적용된다.
1. LESS
    - 브라우저에 내장된 JS인터프리터만으로 컴파일 가능하므로 디펜던시에서 자유로운 것이 특징인 전처리기이다.(*.less)
    - [사이트 바로가기](https://github.com/less/less.js)
2. SCSS
    - Sass 문법을 조금 더 개선하여 보다 쉽게 코딩될 수 있는 문법으로 구성된 전처리기이다.(*.scss)
    - [사이트 바로가기](https://github.com/sass/sass)
3. Sass
    - 가장 오래된 전처리기. 활발히 개발되고 있으며, 가장 많은 개발자들이 선택한 전처리기이다.(*.sass)
    - [사이트 바로가기](https://sass-lang.com/)
4. Stylus
    - 상대적으로 프로그래밍 언어의 특징을 많이 포함하고 있으며, CSS 프로퍼티 내에서 연산자나 함수, 루프 등을 비교적 자유롭게 사용할 수 있고, 반대로 이러한 특징때문에 문법에 혼재가 있어서 처음 전처리를 시작하는 사람에게는 상대적으로 장벽이 높은 편인 전처리기이다.
    - [사이트 바로가기](http://stylus-lang.com)
5. PostCSS
    - Sass, Less 와 같은 전처리기는 내부에서 제공하는 문법이 정해져있고 한정적이지만 PostCSS는 플러그인이 굉장히 다양하다. 이 때문에 최근 Sass와 Less보다 PostCSS사용에 대한 높은 만족도로 선호도가 올라가고 있다.
    - [사이트 바로가기](https://postcss.org)


## HTML 언어의 Prepocessor(전처리기)
CSS가 전처리가 있는 것 처럼 HTML의 요소도 반복, 조건, 함수 처리 등이 가능하도록 한 언어를 의미하며, 이 또한 컴파일이나 번역시스템에 의하여 HTML로 변환되어 웹 브라우저에서 렌더링할 수 있는 템플릿 언어를 가리킨다.
1. Haml
    - 인라인 코드를 사용하지 않고 모든 웹 문서의 HTML을 명확하고 간단하게 설명하는 데 사용되는 마크업 언어로 대부분의 Ruby on Rails 애플리케이션에 사용되는 템플릿 언어이다. 
    - [사이트 바로가기](http://haml.info)
2. Markdown
    - 존 그루버(John Gruber)와 아론 스워츠(Aaron Swartz)가 만들었으며, 주로 간단한 메뉴얼이나 참고문헌을 쉽게 작성할 때 활용하는 템플릿 언어이자 전처리기이다.
    - [사이트 바로가기](https://guides.github.com/features/mastering-markdown)
3. Slim
    - 어설프게 숨기지 않고 핵심 부분에 대한 구문을 줄이는 것을 목표로 하는 템플릿 언어로서의 전처리기로서 단순함을 유지하도록 하면서 충분히 코드를 읽을 수 있도록 해야하는 것이 특징이다.
    - [사이트 바로가기](http://slim-lang.com/)
4. Pug
    - 예전에 Jade 라고 했으며, Haml의 영향을 크게 받아 자바스크립트로 구현된 Node.js 및 브라우저용 고성능 템플릿 언어로 활용되고 있는 전처리기이다.
    - [사이트 바로가기](https://pugjs.org/api/getting-started.html)
